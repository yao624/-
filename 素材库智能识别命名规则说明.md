# 素材库智能识别命名规则说明

## 1. 目的

素材库上传时，系统需要根据文件名自动识别素材的标签、设计师、创意人等信息。

为了避免识别错乱、同名素材冲突、后续扩展困难，文件名需要遵循统一规则。

## 2. 总体原则

1. 文件名只负责承载识别信息，不作为素材唯一标识。
2. 系统内部必须使用上传文件生成的唯一 `file_key` 或临时文件 ID 区分每一条素材。
3. 即使多个素材文件名完全一样，只要上传时的 `file_key` 不一样，系统也必须能正确区分。
4. 智能识别只解析结构化片段，不依赖整段文件名自由猜测。

## 3. 推荐命名格式

推荐统一使用下面的格式：

```text
素材主体名__tag[标签1,标签2]__designer[设计师1,设计师2]__creator[创意人1,创意人2]
```

例如：

```text
夏季女装主图__tag[夏季,促销,女装]
夏季女装主图__tag[夏季,促销]__designer[张三]
夏季女装主图__tag[夏季,促销]__designer[张三]__creator[李四]
```

说明：

1. `素材主体名` 是给人看的，可自由填写。
2. `__` 作为字段分隔符，便于程序稳定解析。
3. `tag[...]` 表示标签集合。
4. `designer[...]` 表示设计师集合。
5. `creator[...]` 表示创意人集合。
6. 方括号内支持多个值，使用英文逗号 `,` 分隔。

## 4. 当前阶段先落地的规则

当前先实现标签识别，所以现阶段至少支持：

```text
素材主体名__tag[标签1,标签2]
```

例如：

```text
连衣裙海报__tag[夏季,女装,上新]
活动主图01__tag[促销,折扣]
视频封面A__tag[短视频,爆款]
```

## 5. 同名素材处理规则

多个素材文件名相同是允许的，但建议补一个可读序号，方便人工区分。

推荐写法：

```text
产品图01__tag[夏季,促销]
产品图02__tag[夏季,促销]
产品图03__tag[夏季,促销]
```

或者：

```text
产品图A__tag[夏季,促销]
产品图B__tag[夏季,促销]
```

需要注意：

1. 同名不应影响系统识别和保存。
2. 系统提交时必须按照每个文件自己的 `file_key` 保存标签关系。
3. 不能把“文件名”当成素材唯一键，否则同名时会串数据。

## 6. 标签识别规则

### 6.1 基本规则

1. 只识别 `tag[...]` 片段中的内容。
2. 方括号内多个标签使用英文逗号 `,` 分隔。
3. 标签名称必须和标签组件中的现有标签完全匹配。
4. 如果识别出的标签在系统中不存在，则提示“没有这个标签”。
5. 如果存在重名标签，则标记为“待确认”，不自动选中。

### 6.2 示例

可识别：

```text
dog__tag[动物,宠物]
banner01__tag[111,222]
```

不可识别或需人工确认：

```text
dog__tag[动物，宠物]
```

说明：这里用了中文逗号，建议统一改成英文逗号。

```text
banner01__tag[不存在的标签]
```

说明：系统中没有对应标签时，需提示用户。

```text
banner01__tag[111]
```

说明：如果系统里有多个同名 `111`，则不能自动绑定，需要人工确认。

## 7. 后续扩展规则

后续设计师、创意人上线后，沿用同一套结构：

```text
素材主体名__tag[标签1,标签2]__designer[张三,李四]__creator[王五,赵六]
```

识别逻辑建议保持一致：

1. 都只解析各自的结构化片段。
2. 都必须和弹窗组件里的数据做精确匹配。
3. 不存在则提示。
4. 重名则待确认。
5. 最终提交都按 `file_key` 写入对应关系。

## 8. 前端交互建议

上传后，智能识别结果建议按单个文件展示：

1. 已自动识别成功的标签，直接选中显示。
2. 未匹配成功的标签，显示“未找到标签”提示。
3. 重名标签，显示“待确认”提示。
4. 允许用户在提交前手动调整每个文件的标签。
5. 即使批量上传多个同名文件，也必须逐条绑定到对应的 `file_key`。

## 9. 不建议的命名方式

以下方式不建议使用：

```text
夏季海报-标签夏季促销女装
```

问题：

1. 结构不固定，程序不好解析。
2. 标签边界不清晰。
3. 后续加设计师、创意人时容易混乱。

以下方式也不建议：

```text
111_222_333
```

问题：

1. 无法判断哪些是标签，哪些是普通文件名内容。
2. 容易误识别。

## 10. 结论

推荐正式采用以下规则作为素材库智能识别标准：

```text
素材主体名__tag[标签1,标签2]__designer[设计师1,设计师2]__creator[创意人1,创意人2]
```

当前先落地：

```text
素材主体名__tag[标签1,标签2]
```

并且必须遵循以下底线：

1. 智能识别靠结构化片段，不靠自由猜测。
2. 同名素材允许存在，但保存时必须按 `file_key` 区分。
3. 标签必须与系统标签库精确匹配。
4. 不存在或重名时，必须提示用户人工确认。
