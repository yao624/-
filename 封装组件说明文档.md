# 封装组件说明文档

本文档放在项目根目录，统一记录项目内已封装的可复用组件。

后续如果继续新增组件，请直接按本文档现有格式往下追加，方便查找、复用、交接。

## 1. 文档使用说明

- 文档用途：统一登记项目内已封装组件的作用、路径、参数、事件和接入方式。
- 适用范围：当前先记录业务侧封装组件，后续公共弹窗、选择器、预览器之类的组件也都可以继续补充进来。
- 维护方式：每次新增或调整封装组件时，同步更新本文件。

## 2. 组件清单

### 2.1 CopywritingPicker 文案选择组件

- 组件名称：`CopywritingPicker`
- 组件路径：`fronted/src/components/copywriting-picker/index.vue`
- 组件类型：弹窗选择器组件

#### 组件用途

用于在业务页面中选择文案。

这个组件只负责：

- 查询文案
- 搜索文案
- 选择文案
- 预览文案内容

这个组件不负责：

- 新增文案
- 编辑文案
- 删除文案
- 管理文案库和文案内容

也就是说，它的定位是“只读选择器”，适合给其他页面嵌入使用。

#### 适用场景

- 广告新建或编辑时，选择已有文案
- 其他业务弹窗中，需要从文案库快速选取文案
- 需要按文案库、文件夹、关键词进行筛选后再选中文案
- 需要预览文案默认内容和多语言内容

#### 主要功能

- 支持切换文案库类型：个人库 / 企业库
- 支持选择具体文案库
- 支持左侧文件夹树筛选
- 支持关键词搜索
- 支持是否包含子文件夹
- 支持单选 / 多选
- 支持默认选中项回显
- 支持右侧预览默认文案内容
- 支持右侧预览多语言内容
- 支持按国家码过滤预览内容

#### Props

| 参数名 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `open` | `boolean` | 无 | 控制弹窗显示与隐藏 |
| `title` | `string` | `选择文案` | 弹窗标题 |
| `width` | `string \| number` | `1380` | 弹窗宽度 |
| `multiple` | `boolean` | `false` | 是否支持多选 |
| `allowEmpty` | `boolean` | `true` | 是否允许不选直接确认 |
| `defaultSelectedRowKeys` | `Array<string \| number>` | `[]` | 默认选中的行 key |
| `defaultSelectedRows` | `PickerRow[]` | `[]` | 默认选中的行数据 |
| `countryCodes` | `string` | `''` | 国家码过滤条件，用于预览时筛选对应语言内容 |

#### Emits

| 事件名 | 参数 | 说明 |
| --- | --- | --- |
| `update:open` | `(value: boolean)` | 同步弹窗开关状态 |
| `cancel` | 无 | 点击取消时触发 |
| `confirm:items-selected` | `(keys, rows)` | 点击确认后返回已选择的文案数据 |

#### 返回数据说明

`confirm:items-selected` 会返回两个参数：

1. `keys`
   当前选中的行主键数组
2. `rows`
   当前选中的完整行数据数组

其中行数据类型为：

```ts
type PickerRow = MetaCopyItemRow & {
  resolved_locale?: string | null;
};
```

也就是说，外部页面在接收后，可以直接拿到文案主数据以及当前解析出的语言信息。

#### 组件内部已接入的能力

该组件当前内部已对接以下文案接口能力：

- 获取文案库列表
- 获取文件夹树
- 获取文案列表
- 获取语言选项

相关接口来源：

- `@/api/meta-copy-library`

#### 当前页面使用示例

参考页面：

- `fronted/src/views/copywritings/index_v2.vue`

示例用法：

```vue
<copywriting-picker
  v-model:open="copywritingPickerOpen"
  :multiple="false"
  :country-codes="previewCountryCodes"
  @confirm:items-selected="handlePickerSelected"
/>
```

#### 接入建议

如果其他页面要复用这个组件，建议按下面方式接：

1. 页面内维护一个弹窗开关变量，例如 `copywritingPickerOpen`
2. 根据业务决定是否单选或多选
3. 如果需要按国家查看预览内容，就传入 `countryCodes`
4. 在 `confirm:items-selected` 事件里接收选中结果并回填到当前表单

#### 注意事项

- 这是“选择组件”，不是“管理组件”，不要在这里继续加新增、编辑、删除文案的能力。
- 如果后续需要做“文案管理弹窗”，建议单独封装新组件，不要和当前选择器混在一起。
- 如果后续业务对返回字段有额外要求，优先在外层页面做兼容处理，避免把选择器改得过重。
- 如果后续要支持更多筛选条件，建议继续沿用当前“左侧树 + 顶部筛选 + 右侧预览”的结构。
